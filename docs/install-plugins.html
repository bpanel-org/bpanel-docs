Installing Plugins

The plugin system built into bPanel is the best and most flexible way to customize your bPanel install while also ensuring compatibility with the platform. Add custom skins, add functionality, or extend existing views. All with a couple simple steps!

With npm

The best way to install plugins is with npm. You can search for any plugins on npm tagged with the bpanel keyword by running

npm search bpanel

Then when you see you a plugin you want to install, just add the name of your plugin to your main bPanel config file, by default created in ~/.bpanel/config.js when you run npm install for the first time. (the name should be a string that matches the name of the target plugin as it is identified in the npm registry)

// ~/.bpanel/config.js
export const localPlugins = [];

export const plugins = ['my-plugin'];

export default { localPlugins, plugins };

Locally

For local plugins that you only intend to keep for your own bPanel install, or for testing, just add the folder to the webapp/plugins/local directory and add the name of the plugin as a string (matching the name of the directory exactly) to the localPlugins array in config.js. (You can use bpanel-cli to create a plugin and point it to your local plugin to install it there).

So if your plugin is in the directory webapp/plugins/local/my-local/plugin you would add the following to your config:

// ~/.bpanel/config.js
export const localPlugins = ['my-local-plugin'];

export const plugins = ['my-plugin'];

export default { localPlugins, plugins };

Once you've finished updating your config file, all you need to do is save your changes. Your server will see the change and re-run webpack to add your plugin to the build!

If you want to learn how to develop and publish your own plugins, head on over to our Developers section to learn more.
